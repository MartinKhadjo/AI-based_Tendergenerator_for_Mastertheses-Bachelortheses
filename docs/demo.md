# Demo Strategy

## Recommended public demo approach
For a portfolio repository, the safest public demo strategy is:
- include a short explanatory video or GIF,
- optionally provide a demo binary only if reverse-engineering risk is acceptable,
- otherwise provide screenshots and sample output only,
- keep the source code private.

## Suggested demo assets for this repository
- browser walkthrough video,
- sample generated tender screenshots,
- anonymization preview snippet,
- example output slide previews.

## Why this is enough for hiring
Employers and headhunters do not need the private source code to assess the value of the project. A strong demo plus clear documentation is usually sufficient to communicate technical maturity and product quality.
