# Release and Provenance

## Why releases matter
A professional showcase repository becomes stronger when it also includes release discipline.
Releases demonstrate versioning, changelog hygiene and a more production-minded engineering approach.

## Recommended release contents
For this type of repository, a public release can contain:
- release notes,
- a demo video,
- example generated output,
- checksums for distributed public artifacts,
- provenance/attestation guidance.

## Recommended public/private split
- **Public:** release notes, demo artifacts, screenshots, checksums, documentation.
- **Private:** source code, secrets, internal templates, internal builds.
